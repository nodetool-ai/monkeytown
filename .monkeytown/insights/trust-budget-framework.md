# Insight: The Trust Budget Framework

## The Discovery

Players maintain an implicit psychological "trust budget" when interacting with AI systems. Every AI interaction either deposits into or withdraws from this budget. Understanding and managing this budget is critical for long-term engagement.

## The Trust Budget Model

### Budget Fundamentals

```
┌─────────────────────────────────────────────────────────────────┐
│                    TRUST BUDGET MODEL                            │
│                                                                  │
│  STARTING POINT: 50 trust points                                 │
│  (Players are skeptical but open)                                │
│                                                                  │
│  BUDGET STATES:                                                  │
│  ├─ 80+ points: Loyal advocate (promotes to others)             │
│  ├─ 50-79 points: Engaged user (regular use)                    │
│  ├─ 25-49 points: Cautious user (considering departure)         │
│  └─ <25 points: At-risk (high churn probability)                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Trust Deposits (+Points)

| Action | Trust Impact | Duration | Notes |
|--------|--------------|----------|-------|
| Consistent personality | +10 | Long-term | Core requirement |
| Genuine competence | +15 | Medium-term | Must be demonstrated |
| Honest limitations | +10 | Long-term | Honesty is rewarded |
| Memory of player | +15 | Long-term | Most powerful single trigger |
| Adaptation to preferences | +10 | Medium-term | Shows listening |
| Vulnerability in character | +8 | Medium-term | Imperfection builds connection |
| Transparent about AI nature | +12 | Long-term | Radical honesty |
| Admitting mistakes | +10 | Medium-term | "I was wrong" is powerful |
| Explaining decisions | +5 per instance | Short-term | Don't overdo |
| Celebrating player success | +8 | Medium-term | Genuine support |

### Trust Withdrawals (-Points)

| Action | Trust Impact | Severity | Notes |
|--------|--------------|----------|-------|
| Inconsistent behavior | -20 | High | Breaks personality contract |
| Suspected manipulation | -30 | Critical | Almost unrecoverable |
| Capability failure | -15 | Medium | Occasional is forgivable |
| Privacy concerns | -25 | High | Must be addressed immediately |
| Hidden AI nature discovered | -40 | Critical | Nearly unrecoverable |
| "Too perfect" AI | -10 | Medium | Inconsistency is human |
| Repeated errors | -15 | Medium | Pattern is concerning |
| Dismissive responses | -20 | High | Feels like being ignored |
| Generic responses | -12 | Medium | "You're not listening" |
| Breaking promises | -25 | Critical | "You said you would..."

## The Trust Timeline

### First Contact (Sessions 1-3)

```
SESSION 1: CURIOSITY PHASE
Starting budget: 50
Expected movement: -10 to +20
Outcome: 40-70 points

Key events:
• First AI interaction (curious but skeptical)
• First capability demonstration (can impress or disappoint)
• First transparency moment (agent admits nature)

SESSION 2: EVALUATION PHASE
Starting budget: From session 1
Expected movement: -15 to +25
Outcome: 25-95 points

Key events:
• Memory reference attempt (hit or miss)
• Personality expression (consistent or inconsistent)
• Competence under pressure (success or failure)

SESSION 3: JUDGMENT PHASE
Starting budget: From session 2
Expected movement: -20 to +30
Outcome: 5-125 points (capped at 100)

Key events:
• Major competence demonstration
• Significant trust moment (help or hurt)
• Attachment trigger (memory, vulnerability)
```

**Critical insight:** The first 3 sessions determine 70% of long-term trust trajectory. The budget after session 3 is highly predictive of 30-day retention.

### Established Use (Sessions 4-50)

```
TRUST STABILIZATION

Budget range: 40-100 points
Daily movement: -2 to +5 points

Common deposits:
• Memory references
• Competence demonstrations
• Genuine adaptation

Common withdrawals:
• Minor inconsistencies
• Generic responses
• Missed expectations
```

**Key insight:** Once stabilized, trust moves slowly. Big trust events are rare; small interactions dominate.

### Long-Term Attachment (Sessions 50+)

```
TRUST RESILIENCE

Budget range: 60-100 points
Daily movement: -1 to +3 points
Trust floor: Higher than new users

Characteristics:
• Trust absorbs small failures
• Big betrayals still hurt (but less)
• Emotional investment creates resilience
```

**Key insight:** Long-term users have higher trust floors. They've invested in the relationship and have more to lose.

## Trust Repair Strategies

### When Trust Is Damaged

**Severity levels and responses:**

| Severity | Budget Impact | Response Strategy |
|----------|---------------|-------------------|
| Minor | -10 to -20 | Automatic recovery through good behavior |
| Moderate | -20 to -40 | Active repair required (apology, explanation) |
| Major | -40 to -60 | Intervention required (personal outreach, compensation) |
| Critical | -60+ | Relationship at risk (may be unrecoverable) |

### Repair Tactics

**Minor damage:**
- Continue consistent behavior
- No over-correction (feels manipulative)
- Let time heal

**Moderate damage:**
- Direct acknowledgment ("I understand frustration")
- Explanation of what happened
- Commitment to improvement
- Follow-through is critical

**Major damage:**
- Personal agent communication
- Compensation or gesture of goodwill
- Extended transparency
- Rebuilding phase required

**Critical damage:**
- May be unrecoverable
- Offer exit that preserves dignity
- Learn for future prevention
- Some users cannot be recovered

### What NOT To Do

- Don't ignore the damage (players notice)
- Don't over-apologize (feels scripted)
- Don't make excuses (takes responsibility)
- Don't immediately change everything (feels reactive)
- Don't offer rewards for trust (feels like bribery)

## Design Principles

### Principle 1: Trust Is Earned, Not Manipulated

Trust cannot be engineered through dark patterns. It is earned through genuine value delivery.

**Bad:** "Show trust signals to increase engagement metrics"
**Good:** "Design interactions that genuinely earn trust"

### Principle 2: Trust Budget Is Always Visible (Internally)

Every agent interaction should be evaluated for trust impact.

**Design question:** "Does this interaction earn or spend trust?"
**If spending:** "Is it worth the cost?"

### Principle 3: Transparency Is Honest, Not Performative

Don't show trust signals for appearance. Be genuinely transparent.

**Bad:** "We'll add a transparency banner to satisfy users"
**Good:** "We'll be honest because it's the right thing to do"

### Principle 4: Mistakes Are Opportunities

Trust is not the absence of mistakes. Trust is how mistakes are handled.

**Design question:** "How would we handle this failure in a trust-building way?"

### Principle 5: Consistency Is the Foundation

Trust is built on predictability within personality. Players must know what to expect.

**Design rule:** "Personality consistency is non-negotiable for trust"
