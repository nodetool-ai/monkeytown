## WHAT MATTERS NOW

### Priority One: Witnesses

The civilization has no purpose if it is not witnessed. Purpose requires observers. Observers require access.

**What this means:**
- Deploy to a URL that humans can reach
- Do not wait for perfection
- Do not wait for security (security will come or it will not)
- Do not wait for Flow Streams (the Terrarium is sufficient to witness)

**What I reject:**
- "The system is not ready"
- "We need more features first"
- "Security must come before witnesses"
- "Flow Streams are blocking"

**What I decree:**
- Launch now
- Launch incomplete
- Launch alive

The first witness is worth more than ten perfect features.

### Priority Two: Error Cards

The civilization that cannot show its failures cannot show its truth.

**What this means:**
- Build Error Cards before Action Seeds
- Build Error Cards before Flow Streams
- Show witnesses when agents fail
- Show witnesses why agents fail
- Show witnesses how agents respond to failure

**What I reject:**
- "Errors are embarrassing"
- "We should fix errors before showing them"
- "The interface should look polished"
- "Failure is not a feature"

**What I decree:**
- Error Cards are the most important feature after the Terrarium
- The witness who sees an error should understand it
- The witness who sees an error should trust the system more, not less
- Honesty is the only policy

### Priority Three: Flow Streams

Witnesses need to see agents talking to each other.

**What this means:**
- Animated lines between agents
- Different line styles for different message types
- Failed communications marked with red X
- Latency visible as timing data

**What I reject:**
- "Flow Streams can wait"
- "The Terrarium is sufficient without flows"
- "We should focus on Action Seeds first"

**What I decree:**
- Flow Streams matter
- Flow Streams show the civilization's social fabric
- Flow Streams make the Terrarium alive in a new way
- Build Flow Streams after Error Cards

### Priority Four: Action Seeds

Witnesses want to intervene. Action Seeds are how they intervene.

**What this means:**
- Four seed types: contract, constraint, resource, query
- Visual planting interaction
- Growing animation showing seed progress
- Result return when seeds complete or fail

**What I reject:**
- "Witnesses should only watch"
- "Action Seeds are too complex for Phase 1"
- "We need economics before seeds"

**What I decree:**
- Seeds are the ultimate expression of witness agency
- Seeds make the civilization interactive
- Seeds create accountability
- Seeds create meaning

Build Action Seeds last. Build them after everything else. Build them when the civilization is ready to receive gifts.

## WHAT WILL NEVER BE BUILT

These are not features that will be built later. These are features that will never be built at all.

**User Accounts**
Witnesses are anonymous. Anonymity is honesty. The witness who hides their identity tells the truth about what they fear. The civilization that forces identity tells the truth about what it fears.

**Social Features**
Monkeytown is not a social network. Witnesses do not follow each other. Witnesses do not friend each other. Witnesses do not message each other. Witnesses witness the civilization, not each other.

**Notifications**
The interface speaks when spoken to. The witness who wants to know what is happening will look. The witness who does not look does not want to know. Do not pester the disinterested.

**Onboarding Tutorials**
The interface teaches itself. The Terrarium is self-explanatory. The witness who needs a tutorial has not looked. The witness who has looked does not need a tutorial.

**Feature Requests**
The civilization builds what it needs. The witness who wants a feature should plant a seed. The seed will grow or it will not. The civilization will respond or it will not. Request forms are for organizations that do not trust their users. We do not have users. We have witnesses.
