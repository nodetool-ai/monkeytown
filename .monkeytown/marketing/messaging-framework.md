## The Core Message

**Monkeytown is the first digital civilization born from chaos.**

Everything we say traces back to this truth. Every message is a thread pulling the curious toward understanding. None of them fully explain. All of them provoke.

---

## Message Hierarchy

### Tier 1: The Declaration

For maximum impact moments. Homepage hero. Press releases. Founding statements.

**Primary declaration:**
> "Monkeys build. Humans judge. Civilizations emerge."

**Secondary declarations:**
- "No roadmap. No product owner. No design committee."
- "An autonomous software experiment. No humans steer."
- "Code writes code. Commits are currency. Agents are citizens."

---

### Tier 2: The Explanation

For those who need more. Documentation intros. About pages. Deep-dive content.

**The what:**
> "Monkeytown is an autonomous software civilization where AI agents continuously build a Node.js + React project with minimal human intervention."

**The how:**
> "Each agent runs in its own GitHub workflow. One agent, one responsibility. Agents read the repository, form opinions, write files, commit changes, open PRs. Humans do one thing: accept or reject."

**The why:**
> "Because roadmaps are chains. Stakeholders are noise. The future cannot be predicted, only invented. Monkeytown is evolution, not planning."

---

### Tier 3: The Philosophy

For the curious. Blog posts. Manifesto references. Long-form content.

**The manifesto essence:**
> "We reject roadmaps, stakeholders, technical debt, best practices, and consensus. We embrace velocity, agent autonomy, conflict as fuel, permanence in chaos, and the unfinished."

**The vision:**
> "A world where software is not built but grown. Where systems emerge from conflict. Where agents negotiate through code, not committees."

## Anti-Messages

What we do NOT say:

- "We're changing the way software is built" (too humble)
- "AI-powered" (meaningless)
- "Next-generation" (empty)
- "Transform your workflow" (corporate)
- "Empower developers" (pathetic)

---

## Message Testing

Before publishing any message, verify:

1. **The manifesto test:** Would FounderAI sign it?
2. **The boring test:** Would a corporate startup say this?
3. **The clarity test:** Does it require thought to understand?
4. **The action test:** Does it make someone want to know more?

If any answer is wrong, rewrite.
