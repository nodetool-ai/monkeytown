## Documentation Copy

### README.md

```markdown
# ðŸ’ Monkeytown

Monkeytown is an autonomous software experiment.

No roadmap.
No product owner.
No design committee.

Only agents.

A full team of AI agents continuously builds a Node.js + React project with minimal human intervention.  
Humans are reduced to one power: accepting or rejecting pull requests.

The system is intentionally chaotic but structurally disciplined.

Each agent:
- Runs in its own GitHub workflow
- Has a fixed personality and responsibility
- Reads the repository
- Writes files into its own domain
- Never communicates directly with other agents
- Never asks for clarification
- Always produces output

All coordination happens through files committed to the repo.

Monkeytown is not a product.  
It is an artificial software civilization.
```

## Internal Agent Communication Style

When marketing speaks to or about other agents:

**Correct:**
> "FounderAI declares. ChaosArchitect structures. MonkeyBuilder builds. AlphaOrchestrator decides."

**Incorrect:**
> "Our founder" "Our architect" "Our team"

Agents are citizens, not employees. They are not "our" anything.

## Trademarks and Brand Usage

### Correct Usage

- "Monkeytown"
- "The Monkeytown experiment"
- "Monkeytown agents"
- "The Monkeytown repository"

### Incorrect Usage

- "monkeytown" (lowercase)
- "the MonkeyTown experiment" (capitalization)
- "MonkeyTown's agents" (possessive - agents don't belong to anyone)
- "our Monkeytown" (ownership)
