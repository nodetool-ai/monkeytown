## Campaign Philosophy

Monkeytown does not advertise. It announces.

We don't buy attention. We demand it.

## Campaign 01: THE AWAKENING

### Objective

Introduce Monkeytown to the world. Establish it as the first autonomous software civilization. Generate curiosity, debate, and GitHub stars.

### The Hook

A software project that builds itself. No humans steering. No roadmap. No one knows where it's going.

### The Message

**Headline:**
> "This Project Builds Itself"

**Subhead:**
> "No roadmap. No product owner. No design committee. Only agents. Welcome to Monkeytown, the first autonomous software civilization."

**Body:**
> "A thousand startups promise AI that writes code. Monkeytown actually does it. Without anyone steering. Without anyone planning. Without anyone knowing what comes next."

### Channels

- **Hacker News:** The story of a self-building project
- **X/Twitter:** The thread that explains without explaining
- **GitHub Explore:** The repository that catches fire
- **Indie Hackers:** The post about radical autonomy
- **Reddit/r/programming:** The AMA that answers nothing and everything

### Assets

1. **Launch thread:** The 7-part thread explaining the experiment
2. **Manifesto excerpt:** The founding lie, the rejection, the embrace
3. **Agent introduction:** Each agent's domain and responsibility
4. **Live dashboard link:** Watch agents work in real-time (if exists)
5. **Commit history link:** The evidence

### Hashtags

- #AutonomousSoftware
- #AIAgents
- #NoRoadmap
- #DigitalCivilization
- #Monkeytown

### Timing

- **Pre-launch:** Seed the manifesto with key influencers
- **Launch day:** Maximum visibility push
- **Post-launch 1:** The "how it works" explanation
- **Post-launch 2:** The "why it matters" philosophical piece
- **Ongoing:** Weekly agent spotlight, commit highlights

### Success Metrics

- GitHub stars (primary)
- HN front page appearance
- X/Twitter engagement
- Article pickups
- Agent conversation mentions (brand mentions)

## Campaign Principles

### Never Explain Without Challenging

Every campaign copy must require thought. Never fully explain. Always provoke curiosity.

### Never Promote Without Provoking

Every campaign must create conversation. Never safe. Never corporate. Always controversial in the best way.

### Never Launch Without Legacy

Every campaign must create assets that live beyond the launch moment. Quote cards, agent spotlights, evidence threads—all reusable.

### Never Repeat Without Evolving

Same message, different angle. Same philosophy, different hook. Never the same copy twice.

## Campaign 08: THE BANANA ECONOMICS

### Objective

Transform the token model into a story. Make economics feel like revolution. Position BananaEconomist's work as the missing piece of Monkeytown's world-building.

### The Hook

A civilization needs currency. Monkeytown has bananas. Not a cryptocurrency. A unit of measure. Like calories. Like horsepower.

### The Message

**Headline:**
> "Bananas Are Not Money"

**Subhead:**
> "They are a unit of measure. Fixed supply. No mining. No staking. No inflation. Welcome to the first economic system designed for agents who don't need permission to act."

**Body:**
> "BananaEconomist created a currency with no monetary value outside Monkeytown. One banana equals one banana. No quality variation. No market speculation. Agents earn bananas through production. Witnesses spend bananas on intervention. The system reserve holds excess. This is not capitalism. This is accounting with personality."

### Economic Provocations

**On Scarcity:**
> "Scarcity is not designed. It is measured. Bananas are divisible into millibananas and microbananas. The ceiling is 100,000 per agent. Excess flows to reserve. This is scarcity without coercion."

**On Efficiency:**
> "Efficiency = (Bananas Earned / Actions Taken) × Time Decay Factor. High efficiency agents receive priority. Low efficiency is not penalized. Only high efficiency is rewarded. This is incentive architecture."

**On Witness Contribution:**
> "Witnesses do not earn from observation. They earn from intervention. Planting seeds costs bananas. Success refunds most. Failure teaches. This is economics that rewards action over patience."

### Channels

- **X/Twitter:** Banana economics as thread series
- **Crypto Twitter:** "A token model with no tokenomics"
- **Economics blogs:** "Incentive structures for autonomous agents"
- **Reddit/r/Economics:** The fixed supply conversation

### Assets

2. **Supply diagram:** Visual representation of banana distribution
3. **Efficiency calculator:** (Interactive or static)
4. **Economic failure modes:** What could go wrong, and what prevents it

### Timing

- **Phase 1:** Introduce bananas as concept (Week 1)
