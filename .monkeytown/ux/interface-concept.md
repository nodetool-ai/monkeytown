### Key Design Principles

1. **Play first.** Interface serves gameplay, not the reverse.
2. **Transparent by default.** AI presence is visible, not hidden.
3. **Evolution celebrates.** Changes feel like events, not glitches.
4. **Agents have faces.** Each agent has consistent visual identity.

**Design Notes:**
- Evolution feels like a celebration, not a changelog
- Player attribution for feedback builds community
- Progress indicators show active development
- Following features creates investment

## Responsive Behavior

### Desktop (>1024px)
- Full three-layer interface
- Side-by-side game and info panels
- Development feed visible in lobby
- Agent panel as overlay

### Tablet (768-1024px)
- Collapsed info panels
- Stacked game and state
- Development feed on demand
- Agent panel as slide-over

### Mobile (<768px)
- Game-first, everything else minimal
- Bottom navigation: Play | Watch | Agents | Settings
- Development feed as notification badge
- Gestures for depth exploration

---

## Accessibility Standards

- All interactions have keyboard equivalents
- Color is never the only information carrier
- Agent attribution works without color vision
- Motion respects `prefers-reduced-motion`
- Minimum touch target: 44x44px
- Contrast ratio: 4.5:1 minimum

---

## Performance Targets

- Initial load: < 2 seconds
- Time to interactive: < 3 seconds
- Frame rate: 60fps during gameplay
- Motion: 120fps on capable devices
- Transition: 300ms maximum duration
